# Popup 使用手册

## 简介

Popup 是一款轻量级 Markdown 实时预览桌面工具。窗口始终置顶显示，适合边写边看效果的工作流。

---

## 启动方式

### 使用可执行文件（推荐）

```bash
# 双击运行
dist\Popup.exe

# 命令行指定文件
dist\Popup.exe path\to\file.md
```

### 使用源码运行

```bash
# 安装依赖（首次）
pip install -r requirements.txt

# 运行
python md_viewer.py

# 或使用启动脚本
start.bat
start.bat path\to\file.md
```

### 使用安装包

运行 `installer\Popup_Setup.exe` 安装后，可通过开始菜单或桌面快捷方式启动。

---

## 打开文件

有三种方式打开 Markdown 文件：

| 方式 | 操作 |
|------|------|
| 菜单打开 | 文件 → 打开（Ctrl+O） |
| 拖拽文件 | 将文件从资源管理器拖入窗口 |
| 命令行参数 | `Popup.exe file.md` |

**支持的文件类型**：`.md`、`.markdown`、`.txt`、及其他文本文件。

---

## 界面功能

### 菜单栏

#### 文件菜单

| 菜单项 | 快捷键 | 功能 |
|--------|--------|------|
| 打开 | Ctrl+O | 弹出文件选择对话框 |
| 刷新 | F5 | 手动刷新当前文件 |

#### 视图菜单

| 菜单项 | 快捷键 | 功能 |
|--------|--------|------|
| 显示/隐藏目录 | Ctrl+B | 切换左侧目录边栏 |

#### 窗口菜单

提供预设分辨率快速调整窗口大小：

- 400 × 300
- 600 × 400
- 800 × 600（默认）
- 1024 × 768
- 1280 × 720
- 1920 × 1080

#### 日志菜单

| 菜单项 | 功能 |
|--------|------|
| 打开日志目录 | 在资源管理器中打开日志文件夹 |

### 目录边栏

窗口左侧显示当前文档的标题目录：

- 自动提取 Markdown 中的 `#` 至 `######` 标题，按层级嵌套显示
- 点击目录项可跳转到对应位置
- 支持拖动调节宽度
- 通过 Ctrl+B 或视图菜单可折叠/显示

### 状态栏

窗口底部实时显示当前文档统计：

```
字符数（无空格）| 字符数（含空格）| 行数
```

### 窗口置顶

窗口始终显示在其他应用之上，无需手动切换。

---

## 自动刷新

打开文件后，应用会自动监听文件变化：

1. 在编辑器中保存文件
2. Popup 检测到文件修改（300ms 防抖）
3. 自动重新渲染并保留滚动位置

无需手动操作，编辑器保存即可看到最新效果。

---

## Markdown 渲染支持

支持以下 Markdown 语法扩展：

| 语法 | 说明 |
|------|------|
| 表格 | GFM 格式表格 |
| 围栏代码块 | \`\`\` 语法 |
| 代码高亮 | 语法着色 |
| 目录 | `[TOC]` 标记自动生成 |
| 换行转换 | 单个换行转为 `<br>` |

### 链接处理

- 外部链接（http/https）：点击后用默认浏览器打开
- 本地文件链接（file://）：用关联应用打开

## 典型使用场景

### 实时文档预览

1. 用编辑器（VS Code、Notepad++ 等）打开 Markdown 文件
2. 启动 Popup 打开同一文件
3. 将两个窗口并排放置
4. 编辑器中修改保存，Popup 自动刷新

### 快速查看文档

1. 将 `.md` 文件拖入 Popup 窗口
2. 通过窗口菜单选择合适的显示大小
3. 窗口置顶，随时参考

## 日志与调试

日志文件位置：

```
%LOCALAPPDATA%\Popup\mdviewer.log
```

可通过菜单 → 日志 → 打开日志目录快速访问。

---

## 常见问题

### 打开文件后窗口空白

- 确认文件为 UTF-8 编码
- 按 F5 手动刷新
- 查看日志获取错误信息

### 文件修改后不自动刷新

- 按 F5 手动刷新
- 重新打开文件（Ctrl+O）
- 查看日志是否有监听器错误

### 启动缓慢

- 建议使用打包后的 `Popup.exe`
- 查看日志中各阶段耗时定位瓶颈
